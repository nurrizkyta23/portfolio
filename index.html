Insight:

Jivajoy is an AI-based e-commerce platform designed to support postpartum mothers in managing stress, baby blues, and insomnia through herbal aromatherapy products. As the Project Manager, I led a team of four members in defining the project scope, scheduling, and implementing the MVP.

Challenges & Solutions
  • Project Cost Estimation At the start of the project, I faced difficulties in projecting the development cost of the MVP. I addressed this by benchmarking similar projects and categorizing the cost components into fixed and variable groups to enable better cost control.

Impacts
  • I successfully kept the project on track both in terms of budget and timeline, with less than 5% deviation from the initial cost estimate and MVP completion on time within six weeks.
  • The team maintained high productivity through Agile Sprint implementation and weekly progress tracking, which improved work efficiency and reduced interdepartmental bottlenecks.
  • A structured communication strategy with stakeholders enhanced decision-making clarity, minimized delays, and maintained a steady feature development rhythm.
  • This experience honed my skills in cost control, dynamic scheduling, and risk management, while also strengthening my ability to foster cross-functional coordination.

ShrimpFarm — Shrimp Farming Digitalizing Solution

Role: Fullstack Developer

Timeline: Jan – May 2023

Tools:

GitHub Repo
Insight:

ShrimpFarm is a comprehensive digital solution designed to help shrimp farmers monitor and optimize their farming operations. The system focuses on manual data entry for parameters such as water temperature, salinity, and feeding schedules, with MongoDB used as the backend database. It aims to support small to medium-scale farmers in transitioning from traditional paper-based methods to digital recordkeeping.

Challenges & Solutions:
  • No IoT Hardware Integration:Due to hardware limitations, all data was manually input. I built a user-friendly data entry form using Laravel and Bootstrap, allowing farmers to log daily pond data easily through a responsive dashboard.
  • Non-technical User Base: Most users had limited tech experience. The interface was designed to be minimal, with large buttons, simple navigation, and basic visualizations (like line charts and alert colors) to ensure usability without requiring prior training.
  • Flexible Data Structure with MongoDB: MongoDB was used for its flexibility in handling evolving data types. I implemented a clean schema structure, making it easy to store, retrieve, and analyze records for each pond without overcomplication.
Impact:
  • Used in a small pilot by students and local farmers to support manual recordkeeping.
  • Helped users log and review daily operational data more consistently compared to handwritten notes.
  • Raised awareness among farmers about the benefits of digital tools in aquaculture and laid the foundation for future enhancements like analytics or semi-automation.

NestEye - Endemic Bird Detection

Role: Data Modeller, Fullstack Developer

Timeline: Apr – May 2025

Tools:

Visit Website
Insight:

NestEye is a web-based deep learning project built to detect and classify Indonesian endemic bird species using image input. Created as a final project for a computer vision course, the goal was to explore real-world implementation of object detection while experimenting with a new tech stack, including Flask for deployment.

Challenges & Solutions:
  • Overfitting on Limited Dataset: The dataset had class imbalances and limited high-quality images for some bird species. To address this, we performed manual data cleaning, applied data augmentation, and experimented with early stopping and dropout regularization to reduce overfitting.
  • Learning New Stack (Flask): This was our first time deploying a machine learning model using Flask. We faced several integration issues between the backend model and the frontend, but through pair programming and consulting Flask documentation and forums, we managed to successfully launch a working demo.
  • Model Selection & Optimization: We used YOLOv5 for object detection due to its real-time capabilities and decent performance on small datasets. Training required tuning learning rates and anchor boxes to better match bird sizes in images.
Impact:
  • Successfully deployed a working demo accessible via browser, allowing users to upload or stream images for detection.
  • Provided practical experience in end-to-end ML project development and web deployment using a lightweight stack.
  • Strengthened teamwork, model debugging, and backend integration skills.
